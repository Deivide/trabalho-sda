\chapter{Descrição do Sistema}

	\indent O objetivo do SDA é ser uma ferramenta distribuída a qual gerencia dowload e upload de arquivos entre clientes e servidor. 
	
	O SDA possui dois módulos principais SDA Cliente e SDA Servidor, estes serão detalhados nas seguintes seções.
	
\section{Módulos}
		
		Os clientes do SDA que podem acessar as funcionalidades do computador SDA. Responsável por gerenciar as requisições de funcionalidades do sistema tendo a interação com o usuário atráves de uma interface.
		Os serviços a serem acessados pelo SDA cliente são:
		
		\begin{itemize}
			\item Login: O cliente pode fazer seu login e ser autenticado no servidor;
			\item Listar todos Arquivos: Listar todos os arquivos presentes no servidor;
			\item Gravar Arquivos: Gravar Arquivos no repositório do servidor;
			\item Baixar Arquivos: Gravar Arquivos no repositório do próprio cliente;
			
		\end{itemize}
		
		Já o servidor é reponsável pelo gerenciamento do sistema, executando as requisições feitas pelo módulo do Cliente e comunicando-se com os repositórios.
		
		Atividades a serem gerenciadas pelo servidor:
		
		\begin{itemize}
			\item Autenticar Clientes: Verificar autenticidade de usuários conectados através de dados fornecidos por estes clientes;
			\item Permitir a visualização da listagem de todos arquivos;
			\item Gerenciar uploads e downloads;
		\end{itemize}
		
		As operações de inclusão, exclusão e consulta de usuários, estão implementadas através RMI (Remote Method Invocation) e dispõem de algoritmos de criptografia para os dados que trafegam pela rede.
		
	As demais operações do sistema utilizam comunicação UDP ou TCP quando necessitas trafegar entre canais de comunicação.
